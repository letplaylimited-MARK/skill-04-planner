```
→ 输出：PASS/FAIL
```

## 核心概念解读

### AI Skill 系统 vs OpenCode

```
AI Skill 系统（规划层）
├─ S00: Navigator - 需求路由
├─ S01: 翻译官 - 需求转技术规范
├─ S02: 工程规划 - 系统架构设计
├─ S03: 开源侦探 - 库选型
└─ S04: 执行规划官 - 分步实施计划
        ↓ (输出 Markdown 计划)
        ↓
OpenCode（执行层）
├─ 自主循环执行
├─ 调用工具（read/write/bash）
├─ 生成代码
└─ 生成测试和文档
        ↓ (输出代码文件)
        ↓
S05 Validator（验收层）
├─ 功能完整度检查
├─ 代码质量检查
└─ 输出 PASS/FAIL 决策
```


## 常见陷阱和解决方案

| 陷阱 | 症状 | 解决方案 |
|------|------|--------|
| **模型选错了** | 生成质量差 | 升级模型（Gemini → Claude） |
| **Prompt 不清晰** | Agent 循环不停 | 按 Markdown 格式提供清晰计划 |
| **成本爆表** | 账单意外增长 | 设置预算告警，用便宜模型 |
| **代码有 Bug** | 生成的代码不对 | 让 OpenCode 反复修复（多轮） |
| **隐私泄露** | 敏感数据上云 | 用 Ollama 本地部署 |
| **网络不稳定** | 连接超时 | 切换到本地模型 Ollama |
