# Sub-Skill 03 · 操作手册生成器

**所属 Skill**：Skill 04 · Planner  
**版本**：v1.0.0  
**职责**：将拆解后的任务树渲染为完整的、人可执行的操作手册

---

## 手册结构规范

```
操作手册
├── 封面页（项目信息/版本/时间估算）

├── 环境配置章节（P0，独立可验证）
├── Phase 1 手册
│   ├── Task 1.1
│   │   ├── Step P1-T1-S1

│   └── ...

```

## 步骤渲染模板

每个步骤按以下固定格式渲染：

```markdown
#### 步骤 {P-T-S}：{描述}

> ⏱ 预计 {X} 分钟 [估算，请根据实际调整]  
> 🔧 工具：{工具名称}  
> 📋 类型：{command / file_edit / config / verify / manual}

**操作**：

```{language}
{具体命令或操作内容}
```

**预期结果**：  
{预期的输出或系统状态}

**验证成功**：  
```bash
{验证命令}
# 期望输出：{期望值}
```

**如果失败**：  
{错误处理建议}
```

## FAQ 自动生成规则

根据步骤内容，自动生成以下类型的 FAQ：

| 触发条件 | FAQ 类型 | 模板 |
|---------|---------|------|
| 有 pip install 步骤 | 依赖安装失败 | 如何解决 pip 安装错误？ |
| 有 API_KEY 配置 | 认证问题 | API 密钥无效怎么办？ |
| 有 docker 命令 | 容器问题 | Docker 启动失败如何排查？ |
| 有数据库操作 | 连接问题 | 数据库连接失败如何排查？ |
| 有端口配置 | 端口冲突 | 端口被占用如何处理？ |

## 断点恢复指南模板

```markdown
## 断点恢复指南

如果执行中断，无需从头开始。按以下步骤恢复：

### 第一步：查看当前状态

```bash
cat .project_state
```

输出示例：
```
PHASE_0=COMPLETE
PHASE_1=COMPLETE
COMPLETED_AT=2025-01-15T14:30:00+08:00
```

### 第二步：对应恢复入口

| .project_state 显示 | 从哪里继续 |
|---------------------|----------|
| 文件不存在 | 从 Phase 0 开始 |
| PHASE_0=COMPLETE | 从 Phase 1 开始 |
| PHASE_1=COMPLETE | 从 Phase 2 开始 |
| PHASE_N=COMPLETE | 从 Phase N+1 开始 |
| PHASE_LAST=COMPLETE | 项目完成，验收 Skill 05 |

### 第三步：环境恢复

每次重新开始前，确保环境激活：
```bash
cd {项目目录}
source venv/bin/activate    # 激活Python环境
# 验证：python --version 应显示正确版本
```
```

## 手册质量检查清单

生成手册后，自动执行以下质量检查：

| 检查项 | 验证方式 | 期望结果 |
|--------|---------|---------|
| 所有步骤有唯一ID | 扫描重复 ID | 无重复 |
| 所有步骤有验证方式 | 扫描空 verify 字段 | 无空白 |
| 版本号标注 [需用户核实] | 关键词扫描 | 全部标注 |
| 每个 Phase 有检查点 | 计数匹配 | Phase数 = 检查点数 |
| 断点恢复表完整 | 检查所有 Phase 条目 | 无缺失 |
| FAQ 覆盖主要步骤类型 | 关键词匹配 | ≥3 条 FAQ |
